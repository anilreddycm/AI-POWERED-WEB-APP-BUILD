# AI Powered Web App Builder

## Overview

The **AI Powered Web App Builder** is a full‑stack starter kit that leverages generative AI to accelerate the creation of modern web applications. It provides a client‑side UI built with vanilla JavaScript and a lightweight Node.js/Express server, along with scripts that demonstrate how to invoke AI models for code generation, translation, and scaffolding.

## Features

- **AI‑assisted code generation** – ready‑to‑use scripts that call an LLM to turn natural‑language descriptions into functional components.
- **Separate client and server workspaces** – clean folder structure (`client/` and `server/`).
- **Zero‑config development** – `npm install` followed by `npm run dev` starts both front‑end and back‑end with hot‑reloading.
- **Docker support** – optional `Dockerfile` and `docker-compose.yml` for containerised deployment.
- **Extensible design** – modular architecture makes it easy to plug in additional AI services or replace the UI framework.

## Prerequisites

- **Node.js** (v20 or later) with npm
- **Python 3.10+** (if you want to run the optional Python helper scripts)
- **Git** (for version control)
- An **OpenAI/Anthropic API key** (or any compatible LLM endpoint) configured via an environment variable `AI_API_KEY`.

## Installation

```bash
# Clone the repository
git clone https://github.com/yourusername/ai_powered_web_app_builder.git
cd ai_powered_web_app_builder

# Install dependencies for client and server
npm ci               # root package.json installs both sub‑projects

# (Optional) Set up environment variables
copy .env.example .env
# Edit .env to add your AI_API_KEY and any other secrets
```

## Development

```bash
# Start the development server (client + API)
npm run dev
```

- The client will be served at `http://localhost:3000`.
- The API server runs on `http://localhost:5000`.
- Hot‑module replacement updates the UI instantly as you edit files.

## Building for Production

```bash
npm run build   # creates optimized static assets in client/dist/
npm start       # runs the production server
```

## Docker Deployment

```bash
docker compose up --build   # builds and runs both services
```

The application will be reachable at `http://localhost` inside the container network.
